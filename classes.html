template<class Socket>
class session¶

The session class represents an SSH session that can be used to open one or more SSH channels.

Template Parameters:

Socket – The type representing the socket used for communication with a remote host. Typically a boost::asio::ip::tcp::socket

template<class CompletionToken>
auto async_handshake(CompletionToken &&handler)¶

Perform the SSH handshake.

This will trade welcome banners, exchange keys, and setup crypto, compression, and MAC layers. This function call always returns immediately.

See also

libssh2_session_handshake

Note

Regardless of whether the asynchronous operation completes immediately or not, the handler will not be invoked from within this function. Invocation of the handler will be performed in a manner equivalent to using net::post.

Parameters:

handler – The handler to be called when the operation completes. The implementation takes ownership of the handler by performing a decay-copy. The handler must be invocable with this signature:

    std::error_code // Result of operation.
);

channel¶

class channel¶

The channel class represents an open SSH channel for receiving and sending data to the remote server.

Public Functions

template<class MutableBufferSequence>
size_t read_some(const MutableBufferSequence &buffers, std::error_code &ec)¶

Read some data from the channel.

This function is used to read data from the channel. The function call will block until one or more bytes of data has been read successfully, or until an error occurs.

See also

libssh2_channel_read

Note

The read_some operation may not read all of the requested number of bytes. Consider using the net::read function if you need to ensure that the requested amount of data is read before the blocking operation completes.

Parameters:

  • ec – Set to indicate what error occurred, if any.

  • buffers – The buffers into which the data will be read.

Returns:

The number of bytes read.
